# P1解決策: 発見経路の構築 — Requirements

## 解決する問題

**P1: Struggling Momentの検索行動とプロダクトが接続していない**

保護者がStruggling Moment（環境構築できない、英語UIで使えない等）で検索しても、本プロダクトに辿り着けない。ランディングページ、SEO対策、検索インデックスのいずれも存在しない。

## 前提・制約

- ホスティング: exe.dev VM
- アーキテクチャ: クライアント完結（サーバーサイドレンダリングなし）
- 初期ターゲット: **保護者（家庭学習）**
- アプローチ: **LP別ページ**（アプリとは分離）

## 全体構成

```
/           → lp.html（ランディングページ）
/app        → index.html（アプリ本体、既存）
```

LPはPyodideを読み込まず軽量に保つ。アプリへの遷移は明確なCTAボタンで行う。

## R1. ランディングページ（lp.html）

### R1.1 ページ構成

保護者のSwitch Timelineに沿った構成で、上から順に読むと「試してみよう」に辿り着く。

| セクション | 目的 | 対応するForce |
|---|---|---|
| メインビジュアル | 第一印象。「何ができるか」を一言で伝える | Pull |
| こんな経験ありませんか？ | 困りごとへの共感 | Push |
| 特徴 | プロダクトの価値を3〜4点で説明 | Pull |
| 安心ポイント | 保護者の不安を解消 | Anxiety解消 |
| 使い方 | 3ステップで使い方を見せる | Anxiety解消 |
| CTA | アプリへの遷移 | — |

### R1.2 メインビジュアル

- キャッチコピー: 保護者に「これだ」と思わせる一文
  - 例: 「ブラウザだけでPythonが動く。小学生のためのプログラミング練習帳」
- サブコピー: 主な価値を補足
  - 例: 「インストール不要。日本語エラー。ぜんぶひらがな。」
- CTAボタン: 「今すぐ試す」→ `/app` へ遷移
- マスコット画像を配置

### R1.3 「こんな経験ありませんか？」セクション

保護者が実際に困っている場面を列挙する。検索クエリと一致する語句を自然に含める（SEO効果）。

含めるべき困りごと:
- 「Pythonをインストールしようとしたけど、難しくて断念した」
- 「Chromebookだからソフトを入れられない」
- 「オンラインのPython環境は全部英語で、子どもには使えなかった」
- 「エラーメッセージが英語で、何を直せばいいか分からない」

### R1.4 「特徴」セクション

3〜4つのカードまたはアイコン付きリストで表示。

| 特徴 | 説明 |
|---|---|
| インストール不要 | ブラウザを開くだけ。Chromebookでもタブレットでも |
| ぜんぶひらがな | ボタンもメニューもひらがな。小学1年生から使える |
| 日本語エラー | 英語のエラーを子どもの言葉に翻訳。直し方も教えてくれる |
| コードが消えない | 自動保存。ブラウザを閉じても大丈夫 |

### R1.5 「安心ポイント」セクション

保護者向けの信頼構築情報。

- **安全性**: データはお子さまのブラウザ内で処理されます。外部サーバーへの送信はありません
- **本物のPython**: WebAssembly技術（Pyodide）により、標準のPython 3が動作します
- **将来性**: 標準的なPython構文をそのまま学べるため、将来本格的な環境に移行しても知識が活かせます
- **無料**: 完全無料。アカウント登録も不要です

### R1.6 「使い方」セクション

3ステップで視覚的に説明。

1. **開く** — URLにアクセスするだけ（スクリーンショット）
2. **書く** — コードを書く（エディタのスクリーンショット）
3. **実行** — ボタンを押して結果を見る（出力エリアのスクリーンショット）

### R1.7 CTAセクション

- 大きなボタン: 「今すぐ試す」→ `/app`
- 補足テキスト: 「アカウント登録不要・完全無料」

### R2.2 ターゲットキーワード

Struggling Momentの検索行動から逆算した重点キーワード:

| 優先度 | キーワード | 検索意図 |
|---|---|---|
| 高 | Python 小学生 | 子ども向けPython環境を探している |

| 高 | Python ブラウザ | ブラウザで動く環境を探している |
| 高 | Python 日本語 子ども | 日本語UIの子ども向け環境を探している |
| 中 | Chromebook Python | Chromebookで使える環境を探している |

| 中 | プログラミング 小学生 無料 | 無料の学習環境を探している |
| 低 | Scratch 次 Python | Scratchからの移行先を探している |

### R2.5 アプリページ（index.html）のメタタグ追加

既存の`index.html`にも最低限のメタタグを追加する。


`noindex`を設定し、検索流入はLPに集中させる。

## R5. 技術要件

### R5.1 ファイル構成（追加分）

```
├── lp.html             ランディングページ
├── lp.css              LP用スタイル（style.cssとは分離）
├── index.html          アプリ本体（既存、/app に配置）
├── assets/
│   ├── ogp.png         OGP画像
│   ├── screenshot-*.png LPに掲載するスクリーンショット
│   └── snake.png       マスコット（既存）
```

### R5.2 ルーティング

exe.dev VMの静的ファイルサーバーで以下のルーティングを実現する:

- `/` → `lp.html`
- `/app` → `index.html`（既存のアプリ）
- 静的アセットは共通の`/assets/`から配信

### R5.3 パフォーマンス要件

- LPはPyodide・CodeMirrorを読み込まない
- LPの初回表示: 1秒以内（Largest Contentful Paint）
- 外部依存: なし（フォント・アイコンもローカルまたはインライン）
