# 設計 — 「れんしゅうちょうのページ」方式によるほぞん機能

## コンセプト

「Pythonれんしゅうちょう」のメタファーを拡張し、**練習帳に複数のページがある**という形で保存を実現する。ノートをめくるように、書いたコードを切り替えて続きから再開できる。

## UIレイアウト

```
┌─────────────────────────────────┐
│  🐍 Pythonれんしゅうちょう        │  ← ヘッダー
├─────────────────────────────────┤
│ [📄 ページ1] [📄 ページ2] [＋]   │  ← ページタブ（新規追加）
├─────────────────────────────────┤
│                                 │
│    コードエディタ                 │
│                                 │
├─────────────────────────────────┤
│  [じっこう] [ほぞん] [おてほん▼]  │  ← ツールバー
│  [きょうゆう] [わからない！]      │
├─────────────────────────────────┤
│    出力エリア                     │
└─────────────────────────────────┘
```

## ページタブの仕様

### 表示

- エディタの直上に横スクロール可能なタブバーを配置
- 各タブ: ページ名を表示（デフォルト名: 「ページ1」「ページ2」...）
- アクティブなタブは色が変わる（例: 背景白 + 太ボーダー）
- 右端に「＋」ボタン（新しいページを追加）
- タブが多い場合は横スクロール（`overflow-x: auto`）

### タブの操作

| 操作 | 動作 |
|------|------|
| タブをタップ | そのページに切り替え（エディタの内容が変わる） |
| 「＋」をタップ | 新しいページを作成（デフォルトコード入り）し、切り替え |
| タブをダブルタップ | ページ名をインライン編集 |
| タブの「×」 | ページを削除（確認ダイアログあり）。常に表示 |

### タブの縮小表示（Chromeの固定タブ方式）

タブが増えたとき、Chromeの固定タブのようにタブ幅を縮小して対応する:

- **〜5ページ**: フルサイズ表示（ページ名 + ×ボタン）
- **6〜12ページ**: ページ名を省略表示（先頭数文字 + 「...」 + ×）
- **13ページ〜**: アイコンサイズまで縮小（ページ番号のみ表示、ホバー/長押しで名前をツールチップ表示）

### 制約

- 最大ページ数: **32ページ**
- 最小ページ数: **1ページ**（最後の1ページは削除不可）
- ページ名の最大文字数: **20文字**

## データモデル

### localStorage キー設計

```
python-editor-pages       → JSON: ページ一覧メタデータ
python-editor-page-{id}   → string: 各ページのコード
python-editor-output-{id} → string: 各ページの最後の実行結果
python-editor-active      → string: アクティブページのID
```

### ページ一覧メタデータの構造

```json
[
  { "id": "p1", "name": "ページ1", "createdAt": 1712188800000 },
  { "id": "p2", "name": "おみくじ", "createdAt": 1712188900000 }
]
```

### 既存データとの互換性

初回起動時に `python-editor-code`（既存キー）が存在する場合:
1. そのコードを「ページ1」として新データ構造に移行
2. 旧キーを削除
3. 以降は新キー設計で保存

## おてほんとの連携

おてほんサンプルを選んだとき:
1. 現在のページを上書きするか、新しいページに開くかを選べる
2. 確認ダイアログ: 「いまのページにいれる？ あたらしいページにする？」
3. 新しいページに入れる場合: サンプル名がページ名になる
4. 32ページ上限に達している場合: 「いまのページにいれる」のみ選択可能（「あたらしいページにする」はグレーアウト）

## 自動保存

- 現在の動作（キーストロークごとにlocalStorage保存）を維持
- 保存先が `python-editor-page-{activeId}` に変わるだけ
- ページ切り替え時も自動的に現在のコードを保存してから切り替え

## 「ほぞん」ボタン

ツールバーに「ほぞん」ボタンを追加する。実態は自動保存と同じだが、子どもに「保存できた！」という安心感を与えるためのもの。

### 動作

1. ボタンを押す
2. 現在のコードをlocalStorageに保存（自動保存と同じ処理）
3. ボタンが一瞬「✓ ほぞんしたよ！」に変わるアニメーション（1.5秒後に元に戻る）

### スタイル

- 色: パステルグリーン系（保存＝安心のイメージ）
- アニメーション: `prefers-reduced-motion` 対応

## ページ切り替え時の出力エリア

- 出力内容をページごとに保存する（localStorageに `python-editor-output-{id}`）
- ページを切り替えると、そのページの最後の実行結果が復元される
- 未実行のページは出力エリアが空

## 新しいページの初期コード

「＋」ボタンで作成した新しいページには、デフォルトの `print("Hello, Python!")` が入る。

## ページ削除のUndo

ページを削除した直後、出力エリアに「もとにもどす」リンクを5秒間表示する。

- 削除したページのコード・出力をメモリ上に一時保持（localStorageには書かない）
- 「もとにもどす」をタップすると、削除したページを元の位置に復元
- 5秒経過 or 別の操作をしたらUndoリンクは消える
- 一度に保持するのは直近1件のみ

## モバイル対応

画面幅が狭い場合（スマホ等）、タブバーを**折り返し表示**する:

- タブを `flex-wrap: wrap` で複数行に折り返す
- 各タブの最小幅を確保しつつ、行が増えても自然に表示
- 横スクロールは使わない（子どもにわかりにくいため）

## きょうゆうとの関係

「きょうゆう」ボタンの動作は変更なし。現在アクティブなページのコードをURL化する。

「きょうゆう」で作ったURLはページ単位のエクスポートとしても機能する。localStorage消失に備え、大事なコードは「きょうゆう」でURLを控えておくことを案内する（ブックマークバナーと同様の一度きりの案内）。

共有URLから開いた場合:
- 新しいページとしてコードを追加（既存ページは保持）
- ページ名: 「きょうゆうされたコード」
- 32ページ上限の場合: 現在のページに上書きするか確認ダイアログを表示

## スタイリング

### タブバー

```css
/* 方針のみ。実装時に詳細を決める */
.page-tabs {
  /* エディタの上、ツールバーの配色に合わせたパステルカラー */
  /* 横スクロール可能 */
  /* タブは角丸、パステルカラー */
  /* アクティブタブは白背景 + ボーダー強調 */
  /* 「＋」ボタンはタブと同じサイズ感 */
}
```

- 配色: 既存のパステルカラー（水色 `#e8f4fc`、ピンク `#f5c6d0`）に合わせる
- タブの角丸: 上側のみ丸くして「ノートのタブ」感を出す
- フォント: 既存と同じ `sans-serif`、ひらがな表記

## 影響範囲

| ファイル | 変更内容 |
|---------|---------|
| `app/index.html` | タブバーのHTML追加 |
| `app/style.css` | タブバーのスタイル追加 |
| `src/storage.js` | ページ管理のCRUD関数追加、既存データ移行 |

## スコープ外

- デバイス間同期（localStorage依存を維持。「きょうゆう」URLで代替案内）
- ページの並び替え（ドラッグ&ドロップ）
- 全ページ一括エクスポート/インポート
