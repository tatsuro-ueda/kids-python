# バグ: 選択範囲の背景色が表示されない

## げんしょう

エディタでテキストを選択しても視覚的フィードバックがまったく無い。
紫色どころか、何も表示されない。

## げんいん（複合的）

### 原因1: .cm-content の不透明背景が選択レイヤーを隠している

CodeMirror 6 の `drawSelection()` は選択範囲を `.cm-selectionLayer`（`z-index: -1`）として
`.cm-content` の**背後に**描画する。

style.css で `.cm-content` に不透明な背景色を設定している:

```css
#editor .cm-editor .cm-content {
  background: #f0f8ff;
}
```

この不透明な背景が選択レイヤーを完全に覆い隠している。

同時に、`drawSelection()` は `hideNativeSelection` でネイティブの `::selection` を
`transparent !important` に設定しているため、ネイティブ選択も見えない。

結果: カスタム選択もネイティブ選択も両方見えない状態。

### 原因2: CSSセレクタの詳細度不足

EditorView.theme() で設定したカスタムテーマのセレクタが
デフォルトテーマのセレクタより詳細度が低かった（クラス3つ vs 5つ）。

### 原因3: background vs backgroundColor の不一致

デフォルトテーマは `background`（ショートハンド）を使用し、
カスタムテーマは `backgroundColor` を使用していた。
`background` ショートハンドは `background-color` をリセットするため、
仮に詳細度で勝っても `background` に上書きされる場合がある。

## しゅうせい方法

1. style.css の `.cm-content` の `background` を削除し、
   `.cm-editor` 側（既にある `#editor` の背景）で代替する
2. editor.js のセレクタをデフォルトテーマと同じ構造に合わせる（修正済み）
3. プロパティを `background` に統一する（修正済み）

## やさしい解説：3枚のガラス板で理解するバグの全体像

### まずは状況を整理

このサイトには、Pythonのコードを書くエディタがあります。
普通、テキストを選択すると**色がついて「ここを選んだよ」と分かる**はずです。
ところが、何を選んでも**まったく色が変わらない**状態でした。

### たとえ話：3枚のガラス板

> 机の上に**3枚のガラス板**が重なっています。
> - 一番下：**色つきシート**（選択範囲を紫に塗る役割）
> - 真ん中：**ガラス板**（コードの文字が書いてある）
> - 一番上：**ブラウザの標準ハイライト**（普通の青い選択色）

正常なら、下の色つきシートが透けて見えるはずです。

### 原因1（最大の原因）：真ん中のガラスが「不透明」だった

CodeMirrorは選択範囲の色を**一番下のレイヤー**に描きます。
文字が書いてある層の**裏側**に置く仕組みです。

ところが、CSSにこんな設定がありました：

```css
#editor .cm-editor .cm-content {
  background: #f0f8ff;  /* 水色の背景 */
}
```

これは「真ん中のガラス板を**不透明な水色に塗った**」のと同じです。
裏側に紫色のシートがあっても、手前の板で完全に隠れます。

さらにCodeMirrorは「一番上のブラウザ標準の選択色」も透明に消していました。

| レイヤー | 状態 | 見える？ |
|---------|------|---------|
| 一番上：ブラウザ標準の選択色 | 透明に消されている | 見えない |
| 真ん中：コードの文字 | 不透明な水色で塗られている | これしか見えない |
| 一番下：紫の選択シート | 裏側に隠れている | 見えない |

**結果：上も下も見えない = 何も表示されない。**

### 原因2：指示書の「優先順位」で負けていた

CSSには**「どの指示を優先するか」のルール**があります。
指示が具体的なほど、優先されます。

> - 自分の指示：「3年生は紫のTシャツを着てね」（ざっくり）
> - デフォルトの指示：「3年2組の出席番号1〜10番は水色のTシャツを着てね」（具体的）

具体的な方が勝つので、自分の「紫にしろ」が無視されていました。

| 誰の指示？ | 具体さ（クラスの数） | 勝ち負け |
|-----------|-------------------|---------|
| デフォルト | クラス5つ分 | 勝ち |
| 自分で書いた | クラス3つ分 | 負け |

### 原因3：指示の「書き方」が違っていた

CSSには背景色を指定する方法が2つあります：

- **`background`**：背景の**すべて**をまとめて指定
- **`backgroundColor`**：背景の**色だけ**を指定

デフォルト側は `background` を使っていました。
`background` はすべてをリセットするので、
先に `backgroundColor` で色を塗っても**まとめて上書きされて消えます**。

> たとえるなら：
> - 自分：「壁の色を紫にして」（色だけ変更）
> - デフォルト：「壁を全部やり直して水色に」（全部リセット）
>
> 結果、紫は消えて水色になります。

### 経緯まとめ

| ステップ | やったこと | 結果 |
|---------|-----------|------|
| 1回目（3/29） | CSSファイルからJavaScriptに移行した | まだ紫にならない |
| 2回目（今回） | 指示書の具体さ（原因2）と書き方（原因3）を修正 | まだ何も見えない |
| 調査 | 不透明な背景色（原因1）が選択レイヤーを隠していると判明 | これが本丸 |

### どう直す？

真ん中のガラス板を**透明に戻す**のが一番大事。
`.cm-content` の `background: #f0f8ff` を外せば、
裏側の紫が透けて見えるようになる。
