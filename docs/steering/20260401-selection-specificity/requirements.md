# 要求内容

## 概要

CodeMirrorエディタでテキスト選択時の視覚的フィードバック（薄藤色の背景ハイライト）を正常に機能させる。

## 背景

エディタでテキストを選択しても、画面上に何の変化も起きない状態が発生しています。選択範囲のハイライトはコード編集における基本的な視覚フィードバックであり、これが機能しないことはユーザー体験を著しく損なうものです。原因は3つの複合的な問題であることが判明しています（詳細はproblems.mdを参照）。

## ユーザーストーリー

| ユーザー   | Python学習者           | オンラインでPythonを学ぶ初学者          |
| ------ | ------------------- | ----------------------------- |
| ジョブ    | コードを編集する            | エディタ上でコードの一部を選択してコピー・削除・置換する |
| 課題     | 選択範囲が見えない           | テキストを選択しても色が変わらず、どこを選択したか分からない |
| 従来のタスク | 勘で操作する              | 選択範囲が見えないまま、感覚でコピーや削除を行う      |
| 従来のコスト | ミスの頻発・ストレス          | 意図しない範囲を操作してしまい、やり直しが発生       |
| 新しいタスク | 選択範囲が薄藤色でハイライトされる    | 選択した部分が明確に視認でき、そのまま操作する       |
| 新しいコスト | 0秒（視覚フィードバックは即時）    | ストレスなく正確に操作できる                |

## 受け入れ条件（Gherkin形式）

### 選択範囲の背景色表示

```gherkin
Given エディタにPythonコードが入力されている
When  マウスドラッグまたはShift+矢印キーでテキストを選択する
Then  選択範囲の背景色が薄藤色（#d7d4f0）でハイライトされる
```

### フォーカス時・非フォーカス時の区別

```gherkin
Given エディタでテキストが選択されている
When  エディタからフォーカスが外れる
Then  選択範囲の背景色が半透明の薄藤色（#d7d4f080）に変わる
```

### Ctrl+A（全選択）の動作

```gherkin
Given エディタにPythonコードが入力されている
When  Ctrl+A（またはCmd+A）を押す
Then  全テキストが薄藤色の背景でハイライトされる
```

### エディタ背景色との共存

```gherkin
Given エディタの背景色が水色（#f0f8ff）で表示されている
When  テキストを選択する
Then  背景色は水色のまま、選択範囲のみが薄藤色でハイライトされる
```

## 成功指標

- エディタ上でテキスト選択時に薄藤色の背景ハイライトが表示される
- フォーカス中とフォーカス外で視覚的に区別できる
- エディタの水色背景との共存が維持される

## スコープ外

以下はこのフェーズでは実装しません:

- 選択文字列と同じ文字列を他の箇所でもハイライトする機能（highlightSelectionMatches）
- テーマカラーの全面的な見直し
- ダークモード対応
