# 課題: 貼り付けたテキストが選択状態にならない

## げんしょう

エディタにテキストを貼り付けると、カーソルが貼り付けたテキストの末尾に移動するだけで、
貼り付けた範囲が選択状態にならない。

## なぜ困るか

貼り付けた直後に「何が貼り付けられたか」が視覚的に分からない。
選択ハイライト機能（20260401-selection-specificity で修正済み）が
せっかく正常に動いているのに、貼り付け時に活用されていない。

貼り付け直後に選択状態になっていれば:
- 貼り付けた範囲が一目で分かる
- そのまま上書き・削除・インデントなどの操作に移れる

## げんいん

CodeMirror 6 のデフォルト動作として、
貼り付け後はカーソルを挿入テキストの末尾に置くだけで、選択状態にはしない。

## しゅうせい方法

`paste` DOMイベントでフラグを立て、
テキスト挿入後にその範囲を `EditorSelection.range(from, to)` で選択状態にする。
約15行の拡張で実装可能。
