# 要求内容

## 概要

エディタにテキストを貼り付けた際、貼り付けた範囲を自動的に選択状態にする。

## 背景

選択ハイライト機能が正常に動作するようになった（20260401-selection-specificity）ため、
貼り付け直後にも選択状態にすることで視覚的フィードバックを最大限に活用できる。

## ユーザーストーリー

| ユーザー   | Python学習者         | オンラインでPythonを学ぶ初学者          |
| ------ | ------------------- | ----------------------------- |
| ジョブ    | コードを貼り付けて試す      | サンプルコードをコピーしてエディタに貼り付ける     |
| 課題     | 貼り付けた範囲が分からない     | 貼り付け後にカーソルが末尾に行くだけで、何が入ったか見えない |
| 従来のタスク | 目視で確認する           | 貼り付け後にスクロールして範囲を確認する         |
| 従来のコスト | 数秒 + 認知負荷         | 特に長いコードだと把握しづらい              |
| 新しいタスク | 貼り付けると自動で選択される   | 貼り付けた範囲が紫色でハイライトされ、即座に把握できる |
| 新しいコスト | 0秒                 | 視覚的に即時フィードバック                |

## 受け入れ条件（Gherkin形式）

### 貼り付けテキストの自動選択

```gherkin
Given エディタにフォーカスがある
When  Ctrl+V（またはCmd+V）でテキストを貼り付ける
Then  貼り付けたテキスト範囲が選択状態（ハイライト表示）になる
```

### 右クリックメニューからの貼り付け

```gherkin
Given エディタにフォーカスがある
When  右クリックメニューから「貼り付け」を実行する
Then  貼り付けたテキスト範囲が選択状態になる
```

### 既存テキスト選択後の貼り付け（置換）

```gherkin
Given エディタでテキストの一部が選択されている
When  Ctrl+Vでテキストを貼り付ける
Then  選択範囲が貼り付けたテキストに置換され、置換後のテキストが選択状態になる
```

## 成功指標

- 貼り付け直後に貼り付けた範囲がハイライト表示される
- 通常の入力操作（タイピング）には影響しない

## スコープ外

- ドラッグ&ドロップによるテキスト挿入の選択
- アンドゥ/リドゥ後の選択状態
