# 要求内容

## 概要

翻訳フィードバックと国際コミュニティの拠点となるDiscordサーバーを開設し、アプリ内からリンクする。

## 背景

i18n Phase 1 で17言語のLLM翻訳が完了したが、翻訳品質はネイティブスピーカーのフィードバックなしには改善できない。i18n-problems.md (P9, P10) で分析した通り、Discordをコミュニティプラットフォームとして選定済み。アプリ内フッターには `https://discord.gg/TODO` のプレースホルダーが配置済みで、実際の招待リンクへの置き換えが必要。

## ユーザーストーリー

### ストーリー1: ネイティブスピーカーが翻訳の問題を報告する

| ユーザー | バイリンガルのプログラマー・教育者 |
|---|---|
| ジョブ | 母語の翻訳品質を改善したい |
| 課題 | LLM翻訳に不自然な表現がある |
| 従来のタスク | 手段がない |
| 従来のコスト | — |
| 新しいタスク | アプリのフッターからDiscordに参加し、言語チャンネルで報告 |
| 新しいコスト | 5分 |

### ストーリー2: 開発者が翻訳フィードバックを受け取る

| ユーザー | プロダクト開発者 |
|---|---|
| ジョブ | 翻訳の問題を効率的に収集・管理したい |
| 課題 | 70言語のフィードバックを構造的に受け取る仕組みがない |
| 従来のタスク | — |
| 従来のコスト | — |
| 新しいタスク | Discordの報告チャンネルで構造化されたフィードバックを受け取る |
| 新しいコスト | 日次5分のチェック |

## 受け入れ条件（Gherkin形式）

### Discordサーバーの存在

```gherkin
Given Discordアカウントを持つユーザーが
When  招待リンクにアクセスする
Then  「Pythonれんしゅうちょう」サーバーに参加できる
```

### チャンネル構成

```gherkin
Given サーバーに参加したユーザーが
When  チャンネル一覧を見る
Then  announcements、report-translation-issues、translation-general、dev、および主要言語チャンネルが存在する
```

### 翻訳報告テンプレート

```gherkin
Given report-translation-issues チャンネルで
When  フォーラム投稿を作成する
Then  言語・場所・現在の翻訳・問題点・提案する修正のテンプレートが表示される
```

### アプリ内リンク

```gherkin
Given アプリを使っているユーザーが
When  フッターの「翻訳を手伝う / Help translate」リンクをクリックする
Then  Discordの招待ページが開く（有効な招待リンク）
```

## 成功指標

- Discordサーバーが開設され、必要なチャンネルが構成されていること
- アプリ内の `TODO` プレースホルダーが実際の招待リンクに置き換わっていること
- 報告テンプレートが設定されていること

## スコープ外

- Discordボットの開発（Phase 2以降）
- GitHub Issue連携の自動化（Phase 2以降）
- 全17言語分の個別チャンネル作成（Phase 1主要言語のみ）
- コーディネーター制度の設計（Phase 3）
