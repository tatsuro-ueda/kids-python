## 実装後の振り返り

### 実装完了日
2026-03-30

### 計画と実績の差分

**計画と異なった点**:
- 当初 i18next の loadNamespaces / reloadResources で解決しようとしたが、キャッシュ不整合が解消せず、3回の試行で断念
- fetch() 直接取得 + focus イベント遅延ロードという根本的にシンプルなアプローチに切り替えた

### 学んだこと
- i18next の複数ネームスペース（translation + samples）の言語切替時ロードは信頼性が低い
- 「いつ取得するか」のタイミングを変える（言語切替時 → 使う時）ことで問題が消えた
- ユーザーの「おてほんボタンを押した時に取得しては？」という提案がブレークスルーになった
