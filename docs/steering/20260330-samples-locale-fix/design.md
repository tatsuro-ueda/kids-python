# おてほんサンプルの言語切替不具合修正 — Design

## 原因

`i18next.changeLanguage()` 後に `i18next.t("list", {ns: "samples"})` を呼んでも、samplesネームスペースが新言語でロードされておらず、フォールバック言語のサンプルが返されていた。

## 修正方針

i18next経由のサンプル取得をやめ、`fetch()` で直接 `/locales/{lang}/samples.json` を取得する。取得タイミングを「言語切替時」から「おてほんセレクタを開いた時（focus）」に変更。

**変更前:**
```
言語切替 → サンプル一覧を全部作り直す → サンプル選択 → コード挿入
（i18nextのキャッシュ不整合で壊れる）
```

**変更後:**
```
おてほんセレクタをタップ → その時点の言語でfetch → 一覧表示 → コード挿入
（シンプル。確実に正しい言語のファイルが返る）
```

## 変更ファイル

- `src/samples.js`: i18next.t() → fetch() に変更
- `src/main.js`: 初期化時・言語切替時のサンプル構築を削除、focusイベントで遅延取得に変更
- `src/i18n.js`: 不要な `reloadResources` を削除
