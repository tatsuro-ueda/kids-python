# アクセス増加施策 — Problems

更新日: 2026-03-30（多言語対応を踏まえて教師セグメントを拡充）

## 現状の課題

プロダクト（Pythonれんしゅうちょう）はStruggling Momentに対する解決策を備えているが、**そもそもターゲットユーザーがこのプロダクトの存在を知らない**。プロダクトの価値が届かなければ「雇われる」ことはない。

多言語対応により、ターゲットユーザーは日本の保護者・教師だけでなく、**世界中の教育者・保護者**に拡大する。特に「教師」は多言語化によって以下のように多様なセグメントに分かれ、それぞれStruggling Momentや意思決定プロセスが異なる。

### 教師セグメントの多様性（多言語対応後）

| セグメント | 地域例 | Struggling Moment | 意思決定の特徴 |
|---|---|---|---|
| **途上国の公立校教師** | インド、バングラデシュ、ナイジェリア | 母語でPythonを教える教材が存在しない。PCが少なくChromebook/タブレッ��中心 | 政府カリキュラムへの準拠が必須。校長・教育委員会の承認が必要 |
| **先進国の公立校教師** | 日本、韓国、フランス | 英語UIのツールは子どもに使わせにくい。環境構築が教室で現実的でない | 学習指導要領との対応を求める。同僚・管理職への説明が必要 |
| **国際学校の教師** | UAE、シンガポール、インド | 多国籍の生徒がいるため、複数言語に対応したツールが必要 | 個人裁量で導入しやすい。最新ツールに関心が高い |
| **NGO・非営利の教育者** | アフリカ、南アジア、中南米 | プログラミング教育をしたいが予算ゼロ。英語が通じない子どもが多い | 無料であることが大前提。オフラインでも使えるかが重要 |
| **塾・EdTech企業の講師** | インド（WhiteHat Jr等）、中南米 | 英語教材を翻訳して使っているが品質が低い | 差別化のために母語ツールを求める。カリキュラムへの組み込みやすさが鍵 |
| **ホームスクール保護者（教師兼任）** | 米国、英国、オーストラリア | 多言語家庭で、子どもの母語でプログラミングを教えたい | 個人で即座に導入できる。コミュニティの評判を重視 |

## 1. 発見経路の不在

### P1. Struggling Momentの検索行動とプロダクトが接続していない ⚠️ 一部対処済み

ターゲットユーザー（保護者・教師）がStruggling Momentに直面したとき、どのような検索行動を取るかを整理する。

| Struggling Moment | 想定される検索クエリ | 現在の接続状態 |
|---|---|---|
| SM1: 環境構築できない | 「子ども Python 簡単」「Python インストール不要」 | ⚠️ LP+meta descriptionあり。SEO効果は未検証 |
| SM2: Chromebookで使えない | 「Chromebook Python」「ブラウザ Python」 | ⚠️ LP+meta descriptionあり。SEO効果は未検証 |
| SM3: 英語UIで使えない | 「Python 日本語」「子ども向け プログラミング 日本語」 | ⚠️ LP+meta descriptionあり。SEO効果は未検証 |
| SM4: エラーが読めない | 「Python エラー 日本語」「SyntaxError 意味」 | ⚠️ LP+meta descriptionあり。SEO効果は未検証 |
| SM7: コードが消えた | （検索に至らずパニック → 離脱） | — |

**対処済み**: LP作成済み（title, meta description, OGP, 構造化データJSON-LD）。Google検索に対応するための基本要素は揃った。

**残る課題**:
- ~~Google Search Consoleへの登録・インデックス確認がまだ~~ ✅ 2026-03-30 完了（所有権確認済み、サイトマップ送信済み、インデックスリクエスト済み）
- ~~検索順位の計測ができていない~~ ✅ 2026-03-30 計測体制構築完了（seo-tracking.md作成、初回ベースライン記録、週次計測ルーチン定義済み。GSCデータ蓄積待ち）
- ブログ記事やコンテンツマーケティングによる被リンク獲得がない

### P2. 口コミ・共有の仕組みがない ✅ 対処済み

- ✅ SNSシェアボタン追加済み（LINE, X, Facebook）— LP下部とアプリ内の2箇所
- ✅ 共有用の分かりやすいURLでアクセス可能
- ❌ 「子どもの作品を見せる」動線がない（Social Job「親に見て！と見せられる」の未活用）
- ❌ 教師が同僚に紹介するための資料がない

### P3. Passive Lookingフェーズの保護者・教育者にリーチできない

Switch Timelineの「第1の考え」段階にいる保護者・教育者は、まだ具体的な検索行動を取っていない。この層へのリーチ手段がない。

**日本国内**:
- プログラミング教育関連のコミュニティに存在感がない
- 教育系メディア・ブログでの言及がない
- SNSでの情報発信がない

**多言語対応後の国際展開**:
- 各言語圏のプログラミング教育コミュニティにリーチできていない
  - インド: WhiteHat Jr. コミュニティ、DIKSHA教師フォーラム
  - アラビア語圏: Madrasa、各国の教育省フォーラム
  - 中南米: EducarChile、Plan Ceibal（ウルグアイ）等の教育プラットフォーム
- Product Hunt / Hacker News / Reddit 等の国際プラットフォームでの紹介がない
- 各言語のEdTechメディアでの言及がない
- NGO・国際機関（UNESCO、Code.org等）との接点がない

## 2. 信頼構築の不足

### P4. Anxietyを解消する情報が不足 ✅ 対処済み

| Anxiety | 必要な情報 | 現在の提供状況 |
|---|---|---|
| 「本当にPythonが動くの？」 | 技術説明（Pyodide/WASM） | ✅ safetyセクションに記載 |
| 「将来移行できる？」 | 標準Python構文である旨の説明 | ✅ safetyセクションに記載 |
| 「データが消えない？」 | 自動保存の説明 | ✅ feature-cardに記載 |
| 「安全なサイト？」 | サーバー不要・データ送信なしの説明 | ✅ safetyセクションに記載 |
| 「誰が運営しているの？」 | 運営者情報 | ✅ フッターに記載（Feel Physics, 連絡先, ポリシー） |

**残る課題**:
- 「なぜ無料か」の説明がない（保護者は「無料=怪しい」と感じる可能性）
- FAQ形式での疑問解消がない

### P5. 教師にとっての導入根拠がない

教師がクラスで採用するには「なぜこのツールか」を説明できる必要がある。多言語対応後は、教師セグメントごとに必要な導入根拠が異なる。

**全セグメント共通の課題**:
- 導入事例がない（「他に使っている先生がいるか？」に答えられない）
- 「なぜ無料なのか」の説明がない（特に途上国の教師は無料=低品質と捉えるリスク）

**セグメント別の課題**:

| セグメント | 必要な導入根拠 | 現状 |
|---|---|---|
| **途上国の公立校教師** | 政府カリキュラムとの対応（例: インドNEPの「6年生からコーディング必修」）。低スペック端末・低速回線での動作保証 | ❌ なし |
| **先進国の公立校教師** | 学習指導要領との対応。管理者（校長・教育委員会）向けの説明資料 | ❌ なし |
| **国際学校の教師** | 多言語対応の具体的な言語数と品質。IB（国際バカロレア）等との親和性 | ⚠️ 多言語対応中 |
| **NGO・非営利の教育者** | オフライン動作の可否。大人数での利用実績。データプライバシーの保証 | ⚠️ プライバシーは対処済み。オフライン・実績はなし |
| **塾・EdTech講師** | カリキュラムへの組み込み方ガイド。カスタマイズの可否。API連携の有無 | ❌ なし |
| **ホームスクール保護者** | 年齢別の推奨使い方。他の保護者の評判・レビュー | ❌ なし |

**多言語対応が生む新たなチャンス**:
- 「70言語で使える唯一の子供向けPython環境」は国際学校・NGO・途上国教師にとって強力な差別化ポイント
- 各言語版のリリースごとに、その言語圏の教育コミュニティへのリーチ機会が生まれる

## 3. Habitを崩す接点がない

- ✅ 「Scratchの次は？」セクションをLPに追加。Scratchからの自然な移行パスを提示
- ✅ アプリ内に「おてほん」としてScratchブロック対応のPythonコード例を用意

- ❌ 「Scratchとの違い」を視覚的に比較する資料がない

**多言語対応後の拡大機会**:
- Scratchは70言語以上に対応済み → Scratch卒業生は世界中にいる
- 各言語のScratchフォーラム・コミュニティが橋渡しの接点になる
- Hedy（50言語対応の段階的プログラミング言語）も競合兼リファラー候補

### P7. 既存教材との併用価値が見えない

保護者がすでに購入した書籍やオンライン教材との関係性が不明。

- 「この本の演習をここで試せます」のような連携がない
- 教科書準拠のサンプルコードがない

### P8. 初回体験後のリテンションがない

Switch Timelineの「最初の体験」で成功しても、再訪問する理由がない。

- 段階的なカリキュラムやチャレンジがない
- 「次に何をすればいいか」のガイダンスがない
- 保護者向けの進捗通知がない

### P9. コミュニティ・フィードバックループがない
