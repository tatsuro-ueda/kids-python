# LPデザイン評価レポート

評価日: 2026-03-29
対象: index.html (Pythonれんしゅうちょう LP)

---

## 1. デザインチェックリスト評価（48 / 100点）

### ビジュアル階層（6/10）

- **良い点:** Hero → 課題提示 → 特徴 → 安心 → 使い方 → CTA の流れは保護者の心理フローに沿っている
- **改善点:**
  - h1(1.8rem) / h2(1.4rem) / h3(1.1rem) のサイズ差が小さく、階層間のコントラストが弱い。h1は2.4rem以上、h2は1.8rem程度ほしい
  - セクション間の視覚的な区切りが弱い。背景色が「白カード」と「ドット背景」の2種類のみで単調

### 配色（5/10）

- **良い点:** #4a90c4（青系）で統一、ドットパターン背景は子ども向けらしい遊び心
- **改善点:**
  - 配色が青一色に偏りすぎ。アクセントカラーがほぼ皆無
  - CTAボタン(#7ec8e3)が背景(#e8f4fc)と同系色で、ボタンが背景に溶け込んでいる。コンバージョンに直結する重大な問題
  - 子ども向けとしては色数が少なく「楽しそう」「カラフル」な印象が弱い

### 余白・レイアウト（7/10）

- **良い点:** max-width: 800px、line-height: 1.7 は適切
- **改善点:** セクション間の余白が控えめで、大胆な空間を取るとスクロール時のリズムが生まれる

### タイポグラフィ（4/10）

- **良い点:** line-height: 1.7 は日本語に適切
- **改善点:**
  - font-family: sans-serif のみで具体的なフォント指定なし。OS/ブラウザで表示が異なるリスク
  - サイズが0.9rem〜1.8remの狭い範囲。LPのh1としては2.5rem以上が一般的
  - 子ども向けの「親しみやすさ」を感じるタイポグラフィ（丸ゴシック等）になっていない

### CTA目立ち度（4/10）

- **良い点:** ファーストビューとページ末尾の2箇所に配置。末尾に「アカウント登録不要・完全無料」のマイクロコピー
- **改善点:**
  - CTAボタン色が背景と同系統で視覚的に浮かない。暖色系（オレンジ、黄色等）にすべき
  - box-shadowがなくフラットで「押せる要素」に見えにくい
  - hover時のインタラクションフィードバック（スケール、色変化等）が弱い
  - CTAが2箇所しかない。Featuresセクション後にも中間CTAが欲しい

### 画像・ビジュアル要素（5/10）

- **良い点:** ヘビのマスコット、3ステップのスクリーンショットにbox-shadow
- **改善点:**
  - マスコットが100pxと小さく、ファーストビューのアイキャッチとして弱い
  - Heroにプロダクトのスクリーンショットやデモ動画がない
  - feature-cardにアイコンやイラストが一切ない。テキストのみのカードは視覚的に退屈

### 社会的証明・信頼要素（2/10）

- **良い点:** 「保護者の方へ」セクション、構造化データ(JSON-LD)
- **改善点:**
  - ユーザーの声（体験談・レビュー）が皆無
  - 利用者数・導入実績・メディア掲載等の数字がゼロ
  - フッターがなく、運営者情報・プライバシーポリシー・問い合わせ先が不在

### モバイル対応（6/10）

- **良い点:** モバイルファースト設計、viewport meta設定済み
- **改善点:**
  - ブレークポイントが768pxの1つだけ
  - フォントサイズのレスポンシブ調整（clamp()等）がない

### ファーストビューの訴求力（5/10）

- **良い点:** プロダクト名が明確、キャッチコピーは簡潔でわかりやすい
- **改善点:**
  - h1の1.8remが小さすぎる
  - プロダクトの実際の画面（ヒーローイメージ）がない
  - 「無料」「登録不要」がファーストビューに明示されていない（末尾CTAにのみ記載）

### 感情的デザイン（4/10）

- **良い点:** ドットパターン、角丸カード、マスコットの存在、「れんしゅうちょう」というネーミング
- **改善点:**
  - 全体的にトーンが落ち着きすぎ。ワクワク感が不足
  - アニメーションが一切ない
  - 「プログラミングで何ができるか」のワクワクする例が提示されていない

## 2. 競合LPとのビジュアル比較

### 比較対象

| サービス | 特徴的なデザイン要素 |
|---|---|
| **Scratch** | 紫基調+カラフルブロック色。ユーザー作品のカルーセル表示。コミュニティ駆動型の社会的証明 |
| **プログラミングゼミ** | オリジナルキャラクター群が豊富。ユーザー作品ギャラリー。保護者/教育者/子どもへの3層メッセージ |
| **Viscuit** | 「コンピュータは粘土だ!!」の強烈なコピー。子ども手描きキャラクターがナビ要素。GIFアニメ使用 |
| **Code.org** | ダークネイビー+紫の高コントラスト。動画コンテンツ。大企業バッキングのスケール感 |

### 自社に足りないもの

1. **キャラクターの存在感** — マスコットが100px・1箇所のみ。競合はキャラクターを各セクションに展開しストーリー性を持たせている
2. **社会的証明** — 利用者の声・利用数・導入事例がゼロ。競合はユーザー作品表示や導入実績をアピール
3. **動画/GIFアニメ** — 静的スクリーンショット3枚のみ。「動く楽しさ」が伝わらない
4. **「楽しさ」の訴求** — コピーが機能説明に偏っている（「インストール不要」「日本語エラー」）。ワクワクする表現がない
5. **色の多様性** — 水色モノトーン。競合はどこもマルチカラー
6. **ターゲット別の情報設計** — 保護者向けセクションが1つあるのみ。教育者向け導線がない
7. **アニメーション/インタラクション** — 動的要素がゼロ

### 自社の強み（守るべきもの）

1. **「本物のPython」という唯一のポジション** — 競合は全てビジュアルプログラミング（ブロック型）
2. **ゼロフリクション** — アカウント登録不要・インストール不要・完全無料
3. **ブラウザ内完結のプライバシー設計** — 外部サーバーへの送信なし
4. **メッセージの明確さ・シンプルさ** — 保護者が知りたい情報が簡潔に伝わる
5. **ページ読み込み速度** — HTML/CSSのみでJSフレームワーク不使用

## 3. ターゲット視点レビュー

### 保護者（意思決定者）視点 — 総合★3.0/5

| 観点 | 評価 | コメント |
|---|---|---|
| 第一印象（信頼感） | ★3 | 清潔感はあるが運営者情報が一切ない |
| 「使わせたい」度 | ★4 | ペインポイント（Chromebook対応等）が的確 |
| 安全性・プライバシー | ★4 | 「ブラウザ内処理」は強いメッセージ |
| 教育的価値の伝わり方 | ★3 | 「本物のPython」は良いがScratchとの位置づけが不明 |
| 無料の説得力 | ★2 | なぜ無料かの説明がなく不信感の余地 |
| 選ぶ理由の明確さ | ★3 | 独自性はあるが競合との差別化を明示していない |
| 情報の充足度 | ★2 | 運営者情報・連絡先・ポリシー・FAQ・対象年齢が欠落 |

**保護者が特に不安に感じる点:**
- 誰が運営しているのか分からない
- 「完全無料」の理由が不明（怪しさの余地）
- 問い合わせ先がない

### 子ども（6-8歳）視点 — 総合★3.3/5

| 観点 | 評価 | コメント |
|---|---|---|
| 見た目の楽しさ | ★3 | テキスト多め、色のバリエーション少ない |
| 「やってみたい！」度 | ★3 | 何ができるかの具体例がない |
| ひらがなの読みやすさ | ★4 | ヒーローはOK。本文は保護者向け（設計として妥当） |
| マスコットの魅力 | ★3 | 100pxで小さい。1箇所しか登場しない |
| 色使い | ★3 | 水色一色で刺激が弱く単調 |
| 怖さ・難しさ | ★4 | 威圧感なし。角丸+柔らかい色で安心感 |

**子どもの視点で特に足りないもの:**
- 「こんなことができるよ！」の具体例
- CTAが子ども向けでない（「今すぐ試す」→「やってみる！」等）
- マスコットのサイズと登場回数
