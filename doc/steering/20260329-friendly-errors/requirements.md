# requirements: エラーメッセージ改善（第2イテレーション）

## たいしょうユーザー

- しょうがっこう1〜2ねんせい（ていがくねん）

## ひょうきルール

- メッセージはすべてひらがな（かんじは使わない）
- コードにでてくるきごう（`)` `'` `:`など）はそのまま表示してよい
- かんすうめい（`str()` `int()` `len()`など）もそのまま表示してよい

## たいしょうエラー

| エラーしゅべつ | たいしょう |
|--------------|----------|
| `SyntaxError` | o |
| `NameError` | o |
| `TypeError` | o |
| `IndentationError` | o |
| `IndexError` | o |
| `ZeroDivisionError` | o |
| `ValueError` | o（`input()` → `int()` のパターンをふくむ）|
| `AttributeError` | o |
| `KeyError` | o |

## メッセージのほうしん

1. **ぐたいてきに**: 「かきかたがまちがっている」ではなく「ここにとじカッコ `)` がないよ」のように、なにをどうすればいいかつたえる
2. **なおしかたをセットで**: 「〜がないよ。〜してね」のように、どうすれば直るかも書く
3. **ぎょうばんごうを目立たせる**: 「○ぎょうめをみてね:」を先頭につける

- `<summary>` のテキストは「くわしくみる」
