# 要求内容

## 概要

LPの画像・ビジュアル要素を強化し、子ども向けプロダクトとしてのワクワク感と保護者への訴求力を高める。CSS/HTMLの変更のみで即実装可能な施策に絞る。

## 背景

LPデザイン評価（doc/design-problems.md）で「画像・ビジュアル要素」が5/10点、「感情的デザイン」が4/10点と低評価だった。特に以下の課題が3つの評価軸（チェックリスト・競合比較・ターゲット視点）で共通して指摘された:

- マスコットが100pxと小さく、ファーストビューのアイキャッチとして機能していない
- Heroセクションにプロダクトの実際の画面がなく「何ができるか」が伝わらない
- feature-cardがテキストのみで視覚的に退屈
- 競合（Scratch, Viscuit, プログラミングゼミ）と比較して色の多様性・動的要素が不足

## ユーザーストーリー

| ユーザー   | 保護者（30代、小学1年生の親）       |
| ------ | ---------------------- |
| ジョブ    | 子どもに合ったプログラミング学習環境を見つける |
| 課題     | LPを見ても実際の画面がわからず、試す決断ができない |
| 従来のタスク | テキスト説明を読んで想像する |
| 従来のコスト | 離脱リスク高（画面イメージなしで判断を求められる） |
| 新しいタスク | ヒーローで実際のアプリ画面を見て直感的に理解する |
| 新しいコスト | 3秒で「子どもでも使えそう」と判断できる |

## 受け入れ条件（Gherkin形式）

### A. マスコット拡大+アニメーション

```gherkin
Given LPをブラウザで開いた
When  ファーストビューが表示される
Then  マスコット画像が150px以上で表示され、ゆっくり上下に浮遊するアニメーションが適用されている
```

```gherkin
Given LPをモバイル（幅375px）で開いた
When  ファーストビューが表示される
Then  マスコットが画面幅に対して適切なサイズで表示され、レイアウトが崩れない
```

### B. Heroにプロダクト画面+背景装飾

```gherkin
Given LPをブラウザで開いた
When  ファーストビューが表示される
Then  アプリの実際の画面（スクリーンショット）がHeroセクション内に表示されている
```

```gherkin
Given LPをブラウザで開いた
When  Heroセクションを見る
Then  背景にグラデーションまたは波形のSVG装飾が表示され、他のセクションと視覚的に差別化されている
```

### C. Feature Card強化

```gherkin
Given LPのfeatureセクションまでスクロールした
When  4枚のfeature-cardが表示される
Then  各カードにテーマに合った絵文字アイコンが表示されている
```

```gherkin
Given LPのfeatureセクションまでスクロールした
When  4枚のfeature-cardが表示される
Then  各カードの上辺に異なるアクセントカラーのボーダーが表示されている
```

```gherkin
Given PC（幅768px以上）でLPを閲覧している
When  feature-cardにマウスカーソルを乗せる
Then  カードがわずかに浮き上がるアニメーション（translateY + shadow変化）が発生する
```

## 成功指標

- デザインチェックリスト再評価で「画像・ビジュアル要素」が7/10以上
- デザインチェックリスト再評価で「感情的デザイン」が6/10以上
- ファーストビューでプロダクトの画面が確認できる（3秒テスト）

## スコープ外

以下はこのフェーズでは実装しない:

- マスコットの表情バリエーション（新規画像生成が必要）
- GIFアニメ/動画の作成
- デバイスモックアップフレーム
- SVGアイコンの新規作成（絵文字で代替）
- 社会的証明セクションの追加
- CTAボタンの色変更（別ステアリングで対応）
- フッターの追加（別ステアリングで対応）
- タイポグラフィの改善（別ステアリングで対応）
